# Phase 3 Verification: 验证指南

## 手动验证步骤

1. **准备工作**
   - 运行 `npm run dev` 启动项目。
   - 打开浏览器，上传一份包含**真实文本**（非纯扫描件图片）的 PDF。
   - 确保你拥有一个可用的 DashScope（阿里云百炼）的 API Key。

2. **验证：文本选中与悬浮按钮**
   - 预期现象：鼠标在 PDF 上的文字区域，光标应变为文本选择图标（`I` 型）。
   - 验证操作：长按鼠标左键拖拽选中文本，松开鼠标后，在选区正上方应弹出一个黑色的“+ 加入摘录池”悬浮按钮。
   - 边界测试：如果在空白处点击，悬浮按钮应立即消失。

3. **验证：加入摘录池**
   - 验证操作：点击“+ 加入摘录池”按钮。
   - 预期现象：
     - 悬浮按钮消失，选中的蓝色高亮效果消失。
     - 右侧工作台自动切换到“摘录池”面板。
     - 刚选中的文字作为一张新卡片出现在右侧，卡片上包含页码、时间、文本内容，以及悬停显示的“删除”按钮。

4. **验证：AI 流式综述生成**
   - 准备操作：在右侧摘录池顶部输入你的 DashScope API Key（`sk-...`），并点击“保存”。
   - 验证操作：确保摘录池内至少有 1 条摘录，然后点击底部的“基于摘录生成综述”。
   - 预期现象：
     - 右侧工作台切换为“AI 综述”模式。
     - 首先出现“正在连接大模型...”的 Loading 状态。
     - 随后文字像打字机一样一个字一个字地输出。
     - 底部按钮变为红色的“停止生成”。

5. **验证：中断生成与重试**
   - 验证操作：在生成文字的过程中，点击“停止生成”。
   - 预期现象：打字机效果立即停止，状态转为 Error（提示生成已中断），并显示“重试”按钮。点击重试，AI 会清空内容重新开始生成。

## 常见错误与排查
- **选中文本时没有任何高亮或按钮**：检查 `usePdf.ts` 中 `TextLayer` 的渲染是否因 API 变更（pdfjs-dist v4）报错。检查 `PdfWorkspace.vue` 的 CSS 中是否加了 `mix-blend-mode: multiply` 以及 `pointer-events: auto`。
- **点击“生成”时卡住或报错**：检查浏览器控制台网络请求，很可能是 API Key 错误、跨域（CORS）问题或网络不通。请确保配置的 Fetch URL 为 `https://dashscope.aliyuncs.com/compatible-mode/v1/chat/completions`。
- **悬浮按钮位置跑偏**：检查 `useSelection.ts` 里的 `rect` 是否通过 `getBoundingClientRect` 计算，并且 `Teleport` 的父级元素没有影响 `fixed` 定位。
