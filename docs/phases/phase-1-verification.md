# 阶段 1 验证方法

## 1. 安装与启动命令
请在项目根目录下运行以下命令：
```bash
# 安装依赖
npm install

# 启动本地开发服务器
npm run dev
```

## 2. 验证步骤
1. **界面加载验证**：打开浏览器（通常为 `http://localhost:5173`），应能看到完整的三栏布局，中间区域显示“欢迎使用 ScholarFlow AI (Vue 3)”的空状态提示。
2. **错误拦截验证**：在左侧区域拖拽或点击上传一张图片（如 `.png` 或 `.jpg`），应在左侧区域显示红色的错误提示“文件不是 PDF 格式”。
3. **上传 PDF 验证**：
   - 拖拽或点击上传一个 `.pdf` 文件。
   - 左侧列表将出现该文档，且默认被选中（蓝色高亮）。
   - 中间区域将短暂显示“正在加载 PDF...”，加载成功后在 Canvas 上渲染出该 PDF 的第一页内容。
4. **多文档切换验证**：
   - 再次上传另一个 PDF 文件。
   - 左侧列表更新，显示两个文档。
   - 点击不同的文档列表项，中间的 PDF 预览区域能正确触发 `usePdf` 的渲染逻辑，切换显示对应的 PDF 第一页。
5. **占位区验证**：右侧面板稳定显示工作台占位及“暂未开启”提示。

## 3. 已知限制
1. 当前仅渲染 PDF 的第一页到 Canvas，没有添加文本层（TextLayer），因此**文本无法选中**（将在阶段 2 完善）。
2. 不支持滚动翻页和视图缩放功能。
3. 没有持久化存储，刷新页面后上传的文档将会丢失（将在后续阶段接入 Dexie）。
4. PDF 解析的错误处理目前比较基础，针对加密文档可能只抛出通用错误。
