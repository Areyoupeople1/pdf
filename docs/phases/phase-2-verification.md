# 阶段 2 验证方法：PDF 分页与缩放 MVP

## 1. 安装与启动命令
请在项目根目录下运行以下命令（若已启动则跳过）：
```bash
npm run dev
```
打开浏览器访问：`http://localhost:5174/`

## 2. 手动验证步骤
### 翻页功能
1. 左侧上传一个有多页的 PDF（如 5 页）。
2. **观察**：中间区域显示第 1 页，顶部工具栏显示 `1 / 5`。
3. **上一页边界**：点击顶部工具栏的“< (上一页)”按钮，此时应为禁用状态（灰色），无法点击。
4. **下一页操作**：点击“> (下一页)”，**预期现象**：页面内容切换，页码变为 `2 / 5`，滚动条自动回到顶部。
5. **下一页边界**：一直点击“>”，直到最后一页，观察“>”按钮是否变灰禁用。
6. **防抖测试**：疯狂快速点击“<”和“>”，**预期现象**：Canvas 上会有短暂的浅色 loading 遮罩层，但浏览器不会崩溃，最终停留的页码应该和上方工具栏数字一致。

### 页码跳转
1. 点击顶部中间的输入框，输入一个有效数字（如 `3`）并按回车。
2. **预期现象**：阅读器直接跳转到第 3 页。
3. 输入一个无效数字（如 `99`）或非数字字符，按回车。
4. **预期现象**：输入框自动恢复为当前合法页码，不会报错。

### 缩放功能
1. 点击“+ (放大)”，**预期现象**：PDF 画面等比放大，下方出现横向/纵向滚动条。
2. 点击“- (缩小)”，直到极限（通常为 50%），观察是否无法再缩小。
3. 点击“重置大小 (旋转箭头图标)”，**预期现象**：瞬间回到 100% 原始大小。

### 切换文档状态重置
1. 当你在当前文档（比如处于第 3 页，放大了 140% 时）。
2. 点击左侧列表切换到另一个刚上传的 PDF 文档。
3. **预期现象**：新文档必须从**第 1 页**开始，且缩放比例回到**默认 100%**。

## 3. 已知限制
1. 当前翻页只支持“点击按钮”或“输入页码”，不支持鼠标中键平滑滚动到下一页（该功能需要在未来阶段结合虚拟滚动与 IntersectionObserver 实现）。
2. 当前仍然没有渲染 `TextLayer`，所以**文本无法被鼠标选中**。这也是我们阶段 3 马上要解决的核心目标。
