# 阶段 5A 手动验证指南：基于当前综述的二次修改

## 验证目的
确认新增的“基于当前综述二次修改”功能正常，不会破坏原有的 `[n]` 引用溯源和高亮体系，且边界状态（如错误恢复、加载拦截）处理正确。

## 验证步骤

### 测试 1：常规二次修改
1. **准备阶段**：
   - 上传一篇包含中英文的 PDF。
   - 提取 2 句中文、1 句英文到右侧摘录池。
   - 点击“基于摘录生成综述”，等待生成带有 `[1][2][3]` 角标的内容。
2. **二次修改**：
   - 在底部的文本框内输入：“请帮我把全文翻译成英文”。
   - 敲击 Enter 键或点击“继续修改”按钮。
   - **预期现象**：
     - 点击发送瞬间，旧的综述**不会立刻消失**，右下角会出现“正在根据您的要求修改中...”的悬浮提示。
     - 稍后，旧的综述被清空，新的英文内容开始像打字机一样流式打印。
     - 文本框变为禁用状态（不可点击）。
     - 等待打印完毕，确认新生成的英文段落末尾仍然带有 `[1]` 这种数字角标。
3. **溯源验证**：
   - 点击新生成的英文段落里的 `[1]`。
   - **预期现象**：左侧 PDF 成功跳到对应页面，并且原文（即使原文是中文的）依然能被成功高亮！

### 测试 2：异常恢复机制
1. 在输入框输入：“把第一段精简一下”。
2. 点击“继续修改”。
3. 在大模型刚刚开始打字（流式返回进行中）时，迅速点击右上角的“停止”按钮！
4. **预期现象**：
   - 流式打印立刻停止。
   - 内容恢复到了修改前的状态（旧版本）。
   - 不会报错崩溃。

### 测试 3：边界拦截测试
1. 在未生成综述（内容为空）时，检查底部的文本框和按钮。**预期现象**：应该是禁用的。
2. 清空文本框内容。**预期现象**：发送按钮应该是禁用的。
3. 在正在生成第一次综述（打字机进行中）时，检查底部的文本框和按钮。**预期现象**：应该是禁用的。
